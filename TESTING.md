# Тестирование системы контекстов событий журнала

## Обзор

Документ описывает процесс тестирования новой системы контекстов событий журнала, которая разделяет обычные события и события Совнгарда.

## Что было реализовано

1. **Модуль валидации metadata** (`lib/godville_sk/game/log_metadata.ex`)
   - Строгая валидация типов событий и контекстов
   - Автоматическое добавление контекста в metadata
   - Нормализация типов данных

2. **Миграция для индексов** (`priv/repo/migrations/20260409201553_add_context_index_to_hero_logs.exs`)
   - GIN индекс для быстрого поиска по любым полям JSON
   - B-tree индекс для фильтрации по контексту
   - B-tree индекс для фильтрации по типу события

3. **Интеграция контекста в hero.ex**
   - Автоматическое определение контекста на основе статуса героя
   - Строгая валидация metadata при добавлении событий
   - Обратная совместимость для старых записей

4. **Фильтрация логов в dashboard_live.ex**
   - Разделение логов по контекстам
   - Отображение только соответствующих событий

## Проверка через IEx консоль

### Проверка контекстов событий

```elixir
# Получите состояние героя
hero_state = GodvilleSk.Hero.get_state("hero_name")

# Проверьте контексты событий
Enum.each(hero_state.log, fn entry ->
  IO.puts("Message: #{entry.message}")
  IO.puts("Context: #{inspect((entry.metadata || %{})[:context])}")
  IO.puts("Type: #{inspect((entry.metadata || %{})[:type])}")
  IO.puts("---")
end)

# Фильтруйте по контексту
normal_logs = Enum.filter(hero_state.log, fn entry ->
  (entry.metadata || %{})[:context] == :normal
end)

sovngarde_logs = Enum.filter(hero_state.log, fn entry ->
  (entry.metadata || %{})[:context] == :sovngarde
end)

IO.puts("Normal logs: #{length(normal_logs)}")
IO.puts("Sovngarde logs: #{length(sovngarde_logs)}")
```

### Проверка валидации metadata

```elixir
# Тест валидации корректного metadata
{:ok, normalized} = GodvilleSk.Game.LogMetadata.validate(%{type: "combat_roll", actor: "hero", target: "enemy", roll: 10, total: 15, damage: 5, is_hit: true}, :normal)
IO.inspect(normalized)

# Тест валидации некорректного metadata
{:error, reason} = GodvilleSk.Game.LogMetadata.validate(%{type: "invalid_type"}, :normal)
IO.inspect(reason)

# Тест валидации с отсутствующими обязательными полями
{:error, reason} = GodvilleSk.Game.LogMetadata.validate(%{type: "combat_roll", actor: "hero"}, :normal)
IO.inspect(reason)
```

SELECT id, message, metadata->>'context' as context, metadata->>'type' as type
FROM hero_logs
WHERE metadata->>'context' = 'sovngarde'
ORDER BY inserted_at DESC
LIMIT 10;

LIMIT 10;

-- Проверьте события по типу

## Возможные проблемы и решения

### Проблема: Индексы не создаются

**Решение:**
```bash
# Проверьте миграцию
mix ecto.migrations

# Повторно запустите миграцию
mix ecto.migrate

# Проверьте индексы в базе
psql -U godville_sk -d godville_sk -h 192.168.61.86 -c "\d hero_logs"
```

### Проблема: События не фильтруются

**Решение:**
1. Проверьте, что миграция выполнена
2. Перезапустите приложение
3. Проверьте логи на наличие ошибок валидации

### Проблема: Старые записи не отображаются

**Решение:**
1. Проверьте функцию `load_from_db` в `hero.ex`
2. Убедитесь, что старые записи получают контекст `:normal`
3. Перезапустите приложение

### Проблема: Ошибки валидации metadata

**Решение:**
1. Проверьте логи приложения
2. Убедитесь, что все события имеют корректный metadata
3. Используйте debug API для проверки валидации

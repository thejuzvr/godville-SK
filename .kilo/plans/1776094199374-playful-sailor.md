# План: Улучшение мозга персонажа для Zero-RPG

## Текущее состояние

Текущая реализация - это простой **State Machine** (конечный автомат):
- Состояния: `idle`, `combat`, `questing`, `traveling`, `tavern`, `camping`, `resting`, `sovngarde`
- Переходы между состояниями основаны на `Enum.random()` - случайные решения
- Нет долгосрочной памяти, настроения, личности

Это неплохо для базовой механики, но для **zero-RPG** (игрок почти не играет, персонаж действует сам) персонаж должен ощущаться "живым".

---

## Предлагаемые улучшения

### 1. Система настроения (Mood System)

Добавить герою эмоциональное состояние, которое влияет на решения и поведение:

```
mood: :neutral | :happy | :angry | :sad | :scared | :excited | :tired
mood_intensity: 0-100 (насколько сильно выражено настроение)
```

**Механика:**
- Победа в бою → `happy`, интенсивность +20
- Поражение → `sad` или `scared`, интенсивность +30
- Низкое HP (< 30%) → `scared`
- Высокий уровень энергии → `excited`
- Много тиков без событий → `tired` растет

**Влияние на поведение:**
- `happy` → более рискованные решения, охотнее идет в бой
- `scared` → избегает боя, ищет отдых
- `angry` → агрессивнее в бою, может нападать первым
- `tired` → приоритет отдыху

---

### 2. Система памяти (Memory System)

Запоминать важные события и использовать их для принятия решений:

```
memory: [
  %{type: :defeat, enemy: "Дракон", location: "Виндхелм", timestamp: ...},
  %{type: :victory, enemy: "Бандит", location: "Рифтен", timestamp: ...},
  %{type: :love_location, location: "Таверна 'Хромая ведьма'", timestamp: ...},
  %{type: :hate_location, location: "Подземелье", timestamp: ...},
]
```

**Использование:**
- После поражения от монстра X → избегать монстров того же уровня
- Помнить любимые локации → чаще туда идти
- Помнить неудачные подземелья → реже выбирать подземелья

---

### 3. Черты характера (Personality Traits)

Каждый герой имеет набор черт, которые влияют на поведение:

```
personality: %{
  risk_tolerance: 0-100,      # Склонность к риску
  greediness: 0-100,          # Сколько любит золото
  bravery: 0-100,             # Смелость в бою
  sociability: 0-100,         # Любит ли компании (таверна)
  curiosity: 0-100,           # Склонность исследовать подземелья
  stubbornness: 0-100         # Склонность не менять решение
}
```

**Влияние:**
- Высокий `greediness` → при полном инвентаре скорее продаст, чем выбросит
- Высокий `bravery` → при низком HP не убегает, а сражается
- Высокий `curiosity` → чаще выбирает подземелья вместо обычных квестов
- Высокий `stubbornness` → завершает квест даже если невыгодно

---

### 4. Система целей (Goal System)

Долгосрочные цели помимо текущих квестов:

```
goals: [
  %{type: :wealth, target: 1000, current: 450, priority: :high},
  %{type: :level, target: 10, current: 5, priority: :medium},
  %{type: :item, target: "Железный меч", found: false, priority: :low}
]
```

**Поведение:**
- `wealth` высокий → при виде монстра с золотом приоритет атаке
- `item` в целях → ищет конкретный предмет в инвентаре/магазине
- Цели влияют на выбор квестов и действий

---

### 5. Умные реакции (Contextual Reactions)

Вместо случайных мыслей - реакции, зависящие от контекста и состояния:

```
Текущее: HP < 30%, в подземелье, mood = scared
Мысль: "Нужно скорее выбраться отсюда, сил почти не осталось..."

Текущее: Победа над боссом, level up, mood = excited  
Мысль: "Я непобедим! Это был эпический бой!"
```

---

### 6. Система страха (Fear System)

Отслеживание опасных ситуаций и избегание:

```
fears: %{
  monster_types: %{"тролль" => 3, "вампир" => 2},  # количество поражений
  locations: %{"Подземелье" => 5},                 # неудачных посещений
  conditions: %{low_hp_combat: 4}                  # опасных ситуаций
}
```

**Поведение:**
- Если `fears["тролль"] > 2` → при встрече с троллем скорее убежит
- Если `fears["Подземелье"] > 5` → избегает подземелий
- Фобии со временем забываются (уменьшаются)

## Реализация

### Структура данных (расширение Hero State)

```elixir
# Добавить в state героя:
mood: :neutral,
mood_intensity: 50,
mood_history: [],           # история изменения настроения

personality: %{
  risk_tolerance: 50,
  greediness: 50,
  bravery: 50,
  sociability: 50,
  curiosity: 50,
  stubbornness: 50
},

memory: [],                 # важные события
fears: %{monsters: %{}, locations: %{}, conditions: %{}},
goals: [],                  # долгосрочные цели

# Таймеры
mood_timer: 0,              # тиков с последней смены настроения
fatigue: 0                  # накопленная усталость
```

### Новые модули

1. `Hero.Mood` - управление настроением
2. `Hero.Memory` - система памяти  
3. `Hero.Personality` - черты характера
4. `Hero.Goals` - система целей
5. `Hero.Brain` - координация всех систем, главный "мозг"

### Интеграция с StateMachine

`StateMachine.tick/1` вызывает `Brain.think/1`, который:
1. Обновляет настроение
2. Анализирует память и страхи
3. Принимает решение с учетом личности и целей
4. Выбирает действие

## Пример "оживления" персонажа

**Сценарий:** Герой с чертами `bravery: 80`, `risk_tolerance: 70`, недавно победил дракона (память), настроение `happy`.

**Без улучшений:**
```
Тик: idle → случайное действие → пошел в подземелье
```

**С улучшениями:**
```
Тик: brain.think()
  - Настроение: happy (интенсивность 70) → рисковать можно
  - Память: недавно победил дракона → "Я сильный!"
  - Цель: level up → подземелье идеально
  - Личность: curiosity 80 → точно иду в подземелье!
  
Мысль: "После той победы над драконом я чувствую себя непобедимым! 
        Самое время исследовать очередное подземелье."
        
Действие: идти в подземелье
```

## Резюме

State Machine - это фундамент, но для zero-RPG нужен "мозг", который:
1. **Помнит** прошлое
2. **Чувствует** настроение  
3. **Имеет характер** (черты личности)
4. **Ставит цели**
5. **Реагирует** умно, а не случайно

Это превратит персонажа из "случайной машины" в "живого героя", за которым интересно наблюдать.

# Проблема зацикливания на таверне - Решение через конечный автомат

## Описание проблемы

Герой постоянно заходит в таверну и выходит из неё в городе:
```
05:31 Зашел в таверну. Решил зайти по привычке.
03:57 Таверна: Выспались в углу. (+15 HP)
03:55 Зашел в таверну. Решил зайти по привычке.
03:16 Таверна: Выпил за компанию. На душе весело!
03:14 Зашел в таверну. Решил зайти по привычке.
```

## Новый подход: гибрид конечного автомата с памятью и рандомом

### Основная логика (гибрид 2 и 3):

1. **Запоминание посещённых городов** - если герой возвращается в тот же город, он не начинает с таверны, а сразу переходит к следующей фазе
2. **Рандом между фазами** - некоторые переходы имеют случайный элемент для вариативности

```
Город (idle) → [Таверна 1 раз, если не был] → [Прогулка/пропуск] → [Квесты] → [Уход]
```

### Правила:

- При первом входе в город: таверна → прогулка → квесты → уход
- При повторном входе в тот же город: прогулка → квесты → уход (таверну пропускаем)
- После прогулки: 70% шанс перейти к квестам, 30% сразу уйти

### 2. Изменить handle_idle_city в state_machine.ex

Полностью заменить логику выбора действия на гибридный автомат с памятью:

```elixir
defp handle_idle_city(state) do
  # Проверяем, был ли уже в этом городе
  was_in_city = state.location in state.visited_cities
  
  case {state.city_phase, was_in_city} do
    # Первое посещение города - сначала таверна
    {:arrived, false} ->
      state
      |> Map.put(:status, :tavern)
      |> Map.put(:city_phase, :tavern_visited)
      |> Map.put(:tavern_visited, true)
      |> Map.put(:tavern_drink_count, 0)
      |> add_to_log("Зашел в таверну. По традиции - сначала сюда.")
    
    # Повторное посещение - пропускаем таверну, сразу прогулка
    {:arrived, true} ->
      state
      |> Map.put(:city_phase, :walked)
      |> Map.put(:walked_in_city, true)
      |> Map.put(:last_action, :walk)
      |> add_to_log("Вернулся в #{state.location}. Осмотрелся - знакомые места.")
    
    :tavern_visited ->
      if state.walked_in_city do
        # Прогулка уже была - переходим к квестам (или уходим с шансом 30%)
        if :rand.uniform() < 0.3 do
          leave_city(%{state | city_phase: :arrived})
        else
          state
          |> Map.put(:city_phase, :quest_checked)
          |> try_take_quest()
        end
      else
        # Прогулка по городу
        state
        |> Map.put(:city_phase, :walked)
        |> Map.put(:walked_in_city, true)
        |> Map.put(:last_action, :walk)
        |> add_to_log("Прогулялся по городу. Осмотрелся.")
      end
      
    :walked ->
      # После прогулки: 70% к кветаm, 30% сразу уйти
      if :rand.uniform() < 0.7 do
        state
        |> Map.put(:city_phase, :quest_checked)
        |> try_take_quest()
      else
        leave_city(%{state | city_phase: :arrived})
      end
      
    :quest_checked ->
      # Квест взят или нет - выходим из города
      state
      |> Map.put(:city_phase, :arrived)
      |> leave_city()
      
    _ ->
      # Фallback - начать заново
      state
      |> Map.put(:city_phase, :arrived)
      |> leave_city()
  end
end
```

### 3. Добавить вспомогательные функции

```elixir
defp try_take_quest(state) do
  # Попробовать взять квест - если есть доступные
  if Enum.random([true, false]) do  # или проверка доступных квестов
    start_quest(state)
  else
    state
  end
end

defp leave_city(state) do
  new_location = GameContent.get_location()
  
  state
  |> Map.put(:location, new_location)
  |> Map.put(:tavern_drink_count, 0)
  |> Map.put(:city_phase, :arrived)
  |> Brain.add_memory(:location_visited, new_location)
  |> add_to_log("Дела в городе сделаны. Отправляюсь в #{new_location}.")
end
```

### 4. Сброс фазы и запоминание города при входе

При перемещении в город (из wilderness) сбрасываем фазу и добавляем в список посещённых:

```elixir
# В handle_status :traveling
defp handle_status(:traveling, state) do
  # ... существующая логика перемещения ...
  
  if GameContent.is_city?(new_location) do
    # Добавляем город в посещённые (с ограничением списка до 10)

    |> Map.put(:city_phase, :arrived)

    |> Map.put(:tavern_visited, false)
    |> Map.put(:walked_in_city, false)
    |> Map.put(:quest_checked, false)
  end
end
```

### 5. Исправить ошибку с неизвестным действием {:auto_equip, ...}

**Проблема:** В `brain.ex:70-71` функция `choose_action` возвращает `{:auto_equip, reason}`, но в `state_machine.ex:770-778` это действие не обрабатывается и падает в fallback на таверну.

**Решение:** Добавить обработку `:auto_equip` в `handle_idle_city`:

```elixir
# Добавить в handle_idle_city перед action ->
{:auto_equip, reason} ->
  # Экипируем лучший предмет из инвентаря
  better_item = find_better_item(state)
  
  if better_item do
    state
    |> Map.put(:equipment, better_item.equipment)
    |> Map.put(:inventory, better_item.inventory)
    |> add_to_log("#{reason} Примерил #{better_item.name}.")
  else
    # Если нечего экипировать - продолжаем городскую логику
    handle_idle_city(%{state | last_action: :auto_equip})
  end

# Также добавить обработку в :tavern_visited и :walked после try_take_quest
# если квест не взят - проверить инвентарь и примерить лучшее
```

## Ожидаемый результат

Герой в городе:

**Первое посещение:**
1. Заходит в таверну (1 раз)
2. Прогуливается по городу (или пропускает с шансом 30%)
3. Если инвентарь полон (80%+) - торговля
4. Проверяет задания (может взять)
5. Уходит из города

**Повторное посещение того же города:**
1. Пропускает таверну
2. Прогулка → торговля (если нужно) → квесты → уход

**Вариативность:**
- 30% шанс пропустить прогулку после таверны
- 30% шанс уйти после прогулки без проверки квестов

**Исправленные ошибки:**
- `{:auto_equip, reason}` теперь корректно обрабатывается (экипировка лучшего предмета)
- Герой больше не зацикливается на таверне

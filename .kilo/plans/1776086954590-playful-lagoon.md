# Расширение логики действий героя

## Цель
Добавить осмысленную систему действий героя, разделенную на городские и полевые активности.

## Текущее состояние
- Локации: простой список названий (`GameData.locations`)
- Состояния: `idle`, `combat`, `questing`, `resting`, `trading`, `sovngarde`
- Действия в idle: случайные (подумать, событие, квест, бой, воровство)

## План реализации

### Этап 1: Система типов локаций

**Файл**: `lib/godville_sk/game_data.ex`

Добавить типы локаций и классификацию:
```elixir
@location_types %{
  "Солитьюд" => :city,
  "Виндхельм" => :city,
  "Вайтран" => :city,
  "Маркарт" => :city,
  "Рифтен" => :city,
  "Данстар" => :city,
  "Винтерхолд" => :city,
  "Морфал" => :city,
  "Фолкрит" => :city,
  # Остальные - :wilderness
}

def location_type(location_name), do: Map.get(@location_types, location_name, :wilderness)
def is_city?(location_name), do: location_type(location_name) == :city
```

**Файл**: `lib/godville_sk/game_content.ex`
Добавить функции-обертки:
- `location_type/1` - вернуть тип локации
- `is_city?/1` - проверить город ли

### Этап 2: Новое состояние :tavern

**Файл**: `lib/godville_sk/game/hero.ex`
Добавить в `defstruct`: `tavern_drink_count: 0` (счетчик выпитого)

**Файл**: `lib/godville_sk/hero/state_machine.ex`

Добавить обработку `:tavern`:
- `handle_status(:tavern, state)` - логика таверны
- Возможные исходы (по завершении переход в :idle):
  - **Напиться** (`drunk`) - шанс разных состояний (веселье, драка, сон)
  - **Выспаться** - восстановление HP
  - **Услышать слухи** - полезная информация (подсказка о монстрах/квесте)
  - **Подраться** - переход в бой

**Новые мысли для таверны** (`GameData`):
- "Этот эль какой-то странный..."
- "Стражник рассказал про бандитов в горах"
- "Кажется, я перепил..."

### Этап 3: Городская логика в :idle

**Файл**: `lib/godville_sk/hero/state_machine.ex`

Обновить `handle_status(:idle, state)`:
```elixir
# Если город - городские действия
# Если не город - полевые действия
```

**Городские действия** (в городе):
1. **Таверна** (20%) - переход в `:tavern`
2. **Прогулка по городу** (25%):
   - Услышать слухи (информация о квесте/монстрах)
   - Торговля - автопродажа инвентаря (если полный)
3. **Взять задание** (25%) - переход в `:questing`
4. **Покинуть город** (30%) - переход в случайную дикую локацию

### Этап 4: Полевая логика

**Новые состояния**:
- `:traveling` - путешествие между локациями
- `:camping` - привал

**Файл**: `lib/godville_sk/hero/state_machine.ex`

Обновить `handle_status(:idle, state)` для дикой местности:

**Полевые действия** (не в городе):
1. **Путешествовать** (35%) - переход в `:traveling`:
   - Выбрать новую локацию
   - Шанс нападения монстра (60%)
   - Шанс случайного события (25%)
   - Шанс бандитов (15%)
2. **Сделать привал** (25%) - переход в `:camping`:
   - Регенерация HP/stamina
   - Шанс ночного нападения (30%)
3. **Найти подземелье** (15%) - переход в квест типа "исследование"
4. **Выполнить задание** (25%) - если есть active квест

### Этап 5: Расширение квестов под подземелья

**Файл**: `lib/godville_sk/game_data.ex`

Добавить типы квестов:
```elixir
@quests [
  %{id: "bounty", type: :bounty, ...},
  %{id: "dungeon", type: :dungeon, steps: 5, reward: ..., dungeon_name: "..."},
  %{id: "delivery", type: :delivery, ...}
]
```

**Логика подземелья** (`handle_status(:questing)`):
- Этапы - это "комнаты" подземелья
- На каждом этапе: шанс ловушки, монстра, сокровища
- Финал: босс или особый предмет

## Приоритеты реализации

# План: Визуальные исправления

## Задачи

### 1. Панель прогресса квеста во время боя

**Проблема**: Панель прогресса квеста скрывается во время боя (находится внутри conditional с combat).

**Решение**: Переместить панель прогресса квеста (`Quest Progress Panel`) из-под блока боя так, чтобы она отображалась всегда, когда герой выполняет квест (`status == :questing`), независимо от того, в бою он или нет.

**Изменения**:
- В `dashboard_live.ex` переместить блок Quest Progress Panel (строки ~279-323) из-под блока `ENEMY COMBAT ARENA` так, чтобы он был над журналом приключений, но показывался когда `status == :questing`
- Условие показа: `@hero_state.status == :questing && @hero_state.target` (уже правильное)

### 2. Замена 3D кубика D20 на прагматичную анимацию

**Проблема**: Текущий 3D кубик выглядит криво и создаёт визуальный мусор.

**Решение**: Заменить 3D кубик на простую анимацию броска с последующим отображением выпавшего числа.

**Изменения в `assets/vendor/d20.js`**:

1. **Упростить DiceRoller class**:
   - Убрать сложную 3D геометрию кубика
   - Вместо этого создать простой круглый контейнер с анимацией "вращения"
   - Показать итоговое число в центре

2. **Новый дизайн**:
   - Круглый контейнер с градиентом (цвет зависит от темы)
   - Анимация: быстрое вращение (1-1.5 сек), затем остановка на числе
   - Число отображается крупным шрифтом в центре
   - Дополнительно: индикатор "герой" или "враг" рядом с числом

3. **Сохранить совместимость**:
   - Метод `roll(number, actor)` продолжает работать
   - Индикатор результата (боевая анимация урона) остаётся

**Визуальный стиль нового кубика**:
```
┌─────────────┐
│  🎲 d20     │
│   ╭───╮     │
│   │ 17 │    │  ← крутится и показывает число
│   ╰───╯     │
│   герой     │
└─────────────┘
```

## Файлы для изменения

1. `lib/godville_sk_web/live/dashboard_live.ex` - переместить Quest Progress Panel
2. `assets/vendor/d20.js` - переписатьDiceRollerclass

## Результат

- Панель прогресса квеста видна всегда, пока выполняется квест
- Кубик D20 - простой, читаемый, без визуальных артефактов

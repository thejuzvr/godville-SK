# Разнообразие квестовой логики

## Цель
Сделать выполнение квестов более интересным и вариативным для наблюдения за прогрессией героя.

## Текущее состояние
- Квесты: просто счетчик прогресса (X/Y)
- Сообщения: "Выполняю кquest: Название (X/Y)"
- Нет разнообразия в зависимости от типа квеста

## План реализации

### Этап 1: Типы квестов с разной механикой

Добавить разные типы квестов в `GameData`:
```elixir
@quests [
  # Исследование подземелий - этапы это комнаты
  %{id: "dungeon_falmer", type: :dungeon, name: "Пещера Фалмеров", steps: 5, reward: 150},
  
  # Охота - убить N монстров
  %{id: "bounty_bandits", type: :bounty, name: "Награда за бандитов", steps: 5, target_count: 5, reward: 100},
  
  # Доставка - путешествие с событиями
  %{id: "delivery_wine", type: :delivery, name: "Доставка вина", steps: 3, reward: 80},
  
  # Сбор - найти предметы
  %{id: "gather_herbs", type: :gathering, name: "Сбор трав", steps: 4, reward: 60}
]
```

### Этап 2: Разнообразные комнаты подземелий

Для квестов типа `:dungeon` - разные события на каждом этапе:
- **Комната с монстром** (30%) - бой
- **Сокровищница** (15%) - получить предмет/золото
- **Ловушка** (15%) - потеря HP, проверить ловкость
- **Отдых** (10%) - восстановить HP
- **Пустая комната** (20%) - просто продвижение
- **Секретная комната** (10%) - бонус

### Этап 3: Обновить handle_status(:questing)

Добавить логику в зависимости от типа квеста:
```elixir
defp handle_status(:questing, state) do
  case state.target.type do
    :dungeon -> handle_dungeon_step(state)
    :bounty -> handle_bounty_step(state)
    :delivery -> handle_delivery_step(state)
    :gathering -> handle_gathering_step(state)
  end
end
```

### Этап 4: Уникальные сообщения

Каждый тип квеста имеет свои сообщения:
- Dungeon: "Исследую комнату...", "Нашел сундук!", "Попал в ловушку!"
- Bounty: "Ищу цель...", "Заметил бандита!", "Найдено 3 из 5"
- Delivery: "Несу посылку в...", "Дорога опасная...", "Достиг пункта назначения"

### Этап 5: Боссы и ключевые моменты

Для подземелий - босс на последнем этапе:
```elixir
# Если это последняя комната подземелья - босс
if progress == total - 1 do
  spawn_boss(state)
else
  random_room_event(state)
end
```

## Примеры сообщений

**Подземелье:**
- "进入新房间: Скелетная камера"
- "Обнаружил сундук! Нашел: Стальной меч"
- "Попал в ловушку! -15 HP"
- "Нашел безопасное место. Отдохнул. +20 HP"
- "Босс! Драугр-военачальник!"

**Охота:**
- "Охочусь на цель 1/5..."
- "Заметил бандита вдалеке!"
- "Найдено 2 из 5. Продолжаю поиск..."

## Приоритеты реализации

1. **Этап 1-2** (типы квестов + комнаты) - основа разнообразия
2. **Этап 3** (логика обработки) - механика
3. **Этап 4** (сообщения) - видимое изменение
4. **Этап 5** (боссы) - кульминация подземелий

## Файлы для изменения

- `lib/godville_sk/game_data.ex` - добавить типы квестов, события комнат
- `lib/godville_sk/game_content.ex` - обертки для новых данных
- `lib/godville_sk/hero/state_machine.ex` - handle_status(:questing) с учетом типа

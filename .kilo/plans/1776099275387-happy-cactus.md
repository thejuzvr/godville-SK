# Plan: Fix Tavern Loop Issue

## Problem
Hero enters and exits tavern repeatedly, getting stuck in a loop. Logs show constant "Зашел в таверну" messages with only brief tavern activities between them.

## Root Cause Analysis
1. No tracking of "last action" - hero has no memory of what it just did
2. `Brain.choose_action/1` in `lib/godville_sk/hero/brain.ex:36` uses weighted decisions without considering recent actions
3. In cities, tavern has base weight of 20 (out of 100), making it frequently chosen
4. After tavern processing completes (status becomes `:idle`), the next decision can be tavern again
5. No penalty for repeating the same action

## Solution

### Step 1: Add `last_action` field to hero state
- Add `last_action: nil` to the hero struct in `lib/godville_sk/hero.ex:16-75`
- Add `last_action` to persistence in `lib/godville_sk/hero/persistence.ex`

### Step 2: Modify weighted_decision to reduce repeat actions
In `lib/godville_sk/hero/brain.ex`, update `weighted_decision/1`:
- Check if `state.last_action` exists
- If the action being considered equals `last_action`, reduce its weight by 50%
- This creates a cooldown effect without eliminating the possibility

### Step 3: Update action handlers to set last_action
In `lib/godville_sk/hero/state_machine.ex`:
- In `handle_idle_city/1` (line 681), set `last_action` when choosing city actions (tavern, quest, trade, walk, leave)
- In `handle_idle_wilderness/1` (line 746), set `last_action` when choosing wilderness actions
- Update `handle_status(:tavern, ...)` (line 270) to preserve last_action or set appropriately
- Update `handle_status(:questing, ...)` (line 323) to track quest as last_action

### Step 4: Handle persistence for last_action
- Ensure `last_action` is properly serialized/deserialized in `lib/godville_sk/hero/persistence.ex`
- Add to both `load_from_db` and `save_to_db` functions

## Files to Modify
1. `lib/godville_sk/hero.ex` - Add last_action field
2. `lib/godville_sk/hero/brain.ex` - Add action cooldown logic in weighted_decision
3. `lib/godville_sk/hero/state_machine.ex` - Set last_action when taking actions
4. `lib/godville_sk/hero/persistence.ex` - Handle last_action persistence

## Expected Result
Hero will have ~50% reduced probability of repeating the same action immediately, leading to more varied behavior in cities (more quests, trading, walking, leaving the city).

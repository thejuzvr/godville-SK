# План: Хранитель боя - новая вкладка журнала приключений

## Цель
Создать новую вкладку "Хранитель боя" для хранения бросков последнего сражения, отдельно от обычного журнала приключений.

## Изменения

### 1. lib/godville_sk/game/log_metadata.ex
- Добавить `:battle_keeper` в список `@contexts` (строка 8)

### 2. lib/godville_sk/hero/state.ex (или combat.ex)
Добавить поле `battle_log` в структуру состояния героя:
- `battle_log: []` - список боевых бросков текущего/последнего боя

Модифицировать логику:
- При начале боя (`combat_initiative`) очищать `battle_log`
- При каждом броске боя (`combat_roll` или `initiative_roll`) добавлять запись в `battle_log`
- При окончании боя (победа/поражение) оставлять `battle_log` нетронутым (это "последнее сражение")

### 3. lib/godville_sk/hero/combat.ex  
Модифицировать функцию `add_to_log` (строка 164):
- Для записей с типом `"combat_roll"` или `"initiative_roll"` использовать контекст `:battle_keeper`
- Также добавлять эти записи в `state.battle_log`
- Для остальных записей (победа, поражение и т.д.) оставить `:normal`

### 4. lib/godville_sk_web/live/dashboard_live.ex
- Добавить assigns для состояния активной вкладки (`:journal` или `:battle_keeper`)
- Добавить функцию `battle_logs(assigns)` которая возвращает `hero_state.battle_log`
- Модифицировать `normal_logs` для исключения боевых бросков (тип "combat_roll" и "initiative_roll")
- Добавить UI переключения вкладок в заголовке журнала
- Показывать соответствующие записи в зависимости от выбранной вкладки

## Результат
- Обычный журнал показывает только небоевые события (мысли, задания, действия)
- Вкладка "Хранитель боя" показывает броски только последнего сражения
- Переключение между вкладками в UI журнала

# План анализа и запуска проекта Godville-SK

## Обзор проекта
**Тип:** Phoenix веб-приложение на Elixir
**Назначение:** Игра "Godville" - idle RPG с героями в мире Elder Scrolls
**Порт:** 5000 (dev)
**База данных:** PostgreSQL

## Архитектура

### Основные компоненты
- **Phoenix Framework** - веб-сервер и LiveView для real-time UI
- **Ecto** - ORM для работы с PostgreSQL
- **GenServer** - управление процессами героев
- **Phoenix.PubSub** - real-time обновления состояния

### Игровая механика
- **Герои:** с атрибутами TES (strength, intelligence, willpower, agility, speed, endurance, personality, luck)
- **D20 система:** броски с модификаторами атрибутов и удачи
- **Статусы:** idle, combat, resting, questing, fleeing, trading, leveling_up
- **Тики:** каждые 2 секунды
- **Сохранение:** каждую минуту в БД

### Логирование
- **В памяти:** до 10 записей в GenServer state героя
- **В БД:** схема `HeroLog` для персистентных логов
- **Отображение:** в `DashboardLive` (журнал приключений)

## План выполнения

### Шаг 1: Проверка зависимостей
- Проверить наличие Elixir и PostgreSQL
- Проверить наличие зависимостей в `mix.exs`
- Убедиться что `mix.lock` существует

### Шаг 2: Настройка базы данных
- Проверить переменную окружения `DATABASE_URL`
- Если не задана - создать локальную PostgreSQL базу
- Выполнить миграции: `mix ecto.migrate`
- Заполнить начальными данными: `mix run priv/repo/seeds.exs`

### Шаг 3: Установка зависимостей
- Выполнить `mix deps.get`
- Установить assets зависимости: `mix assets.setup`
- Собрать assets: `mix assets.build`

### Шаг 4: Запуск приложения
- Запустить Phoenix сервер: `mix phx.server`
- Ожидать запуск на порту 5000
- Проверить логи запуска

### Шаг 5: Мониторинг логов (45 секунд)
- Следить за консольными логами Phoenix
- Идентифицировать:
  - Запуск сервера
  - Подключение к БД
  - Инициализация GenServer процессов
  - PubSub подписки
  - LiveView подключения
  - Игровые тики и события героев
- Зафиксировать любые ошибки или предупреждения

### Шаг 6: Анализ результатов
- Проверить успешность запуска
- Идентифицировать проблемы (если есть)
- Документировать наблюдения
